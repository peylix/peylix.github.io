def towerOfHanoi(disks, initial, transfer, target):

	if disks == 1:

		print("Move disk", disks, "from tower", initial, "to tower", target)

		return

	towerOfHanoi(disks - 1, initial, target, transfer)

	print("Move disk", disks, "from tower", initial, "to tower", target)

	towerOfHanoi(disks - 1, transfer, initial, target)
	

disks = int(input("Give me your number of disks: "))

towerOfHanoi(disks, "A", "B", "C")

The key to the Tower of Hanoi algorithm and any recursion like that is that the function is expected to call itself, which means the same function is also within the function. In logic, this means when the computer execute this piece of code and meet the recursion part, it will execute the “sub-part of the function”.

In line 5 in this piece of code, we let the first variable be “disks - 1”. It its extremely important. With this, the program will continuely execute the code above untill it reach the condition where “disks == 1” which is the end point of the Hanoi function.

Besides, changing the order of variables is also important. It can easily conduct the action of moving a disk from one tower to another. The alteration only happened to the “name” of the variables. **This is because we were not assigning the value A, B and C to the variables, but were only linking them to the “position” of the variables. When the order (or “position”) of these variables changed, for example, from

1

towerOfHanoi(disks - 1, initial, target, transfer)

to

1

towerOfHanoi(disks - 1, transfer, initial, target)

the value A, B and C will be linked to the new order of variables, like from

1

3

initial = "A"
target = "B"
transfer = "C"

to

1
2
3

transfer = "A"
initial = "B"
target = "C"

This is simple. Don’t get confused by it!

Thus, the key is to change the target variable a little bit so that the program can go back and do it again. Considering this feature of recursion, I think it may can serve as an advanced version of loop.

In addition, don’t forget to set an end point! Otherwise the program will be trapped in a loop that will never end!

However, what is more important is the idea that simplifying the tower of Hanoi problem so that it can be express by a small piece of code.
